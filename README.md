# Telegram RAG-бот для курса Big Data (Gemma 2B)

Этот проект представляет собой Telegram-бота, использующего модель Gemma 2B (через Ollama) для ответов на вопросы по курсу Big Data. Бот использует связку LangChain и ChromaDB для реализации технологии RAG (генерация с дополнением извлеченными данными).

## Требования

* Docker и Docker Compose
* Токен Telegram-бота (полученный у @BotFather)
* Python 3.10 или выше (для локального запуска скриптов, если потребуется)

### Регистрация бота (Telegram)

Чтобы получить токен для работы бота:
1. Найдите **@BotFather** в Telegram.
2. Отправьте команду `/newbot` и следуйте инструкциям (нужно выбрать имя и юзернейм).
3. Скопируйте полученный **API Token** и вставьте его в файл `.env` в поле `TELEGRAM_TOKEN`.
4. (Опционально) Используйте команду `/setcommands`, чтобы добавить быстрые команды (`start`, `clear`, `model`) в меню бота.

## Установка

1. Клонировать репозиторий:
   ```bash
   git clone <url-репозитория>
   cd <название-проекта>
   ```

2. Настроить переменные окружения:
   Создайте файл `.env` в корневой папке проекта по следующему образцу:
   ```env
   TELEGRAM_TOKEN=ваш_токен_телеграм
   OLLAMA_HOST=http://ollama:11434
   GEMMA_MODEL=gemma:2b
   ```

3. Подготовить учебные материалы:
   Поместите ваши файлы (.pdf, .txt, .md) в папку `course_docs/`.

## Развертывание через Docker

Для упрощения работы с Docker Compose используется Makefile.

1. Запустить инфраструктуру (Ollama + Бот):
   ```bash
   make run
   ```
   *Примечание: При первом запуске сервис ollama-init автоматически скачает модель Gemma 2B (около 1.6 ГБ).*

2. Проверить готовность модели:
   ```bash
   make test-ollama
   ```

3. Индексировать учебные материалы в векторную базу данных:
   ```bash
   make ingest
   ```

## Использование Makefile

Makefile поддерживает следующие команды для управления проектом:

### Управление контейнерами
* `make run` — собрать образы и запустить все сервисы в фоновом режиме.
* `make stop` — остановить все сервисы.
* `make status` — показать статус контейнеров.
* `make clean` — удалить контейнеры, тома (включая базу данных) и неиспользуемые образы.

### Индексация и данные
* `make ingest` — запустить скрипт загрузки документов из папки `course_docs/`.
* `make re-ingest` — полностью удалить текущую базу данных и заново индексировать все документы.

### Обслуживание и логи
* `make logs` — просмотр логов всех сервисов в реальном времени.
* `make logs-bot` — логи только Telegram-бота.
* `make logs-ollama` — логи сервера Ollama.
* `make build` — принудительная пересборка образа бота без использования кэша.
* `make shell` — открыть интерактивный терминал внутри контейнера бота.

## Техническая архитектура

* **Инференс**: Ollama (локальный запуск Gemma 2B).
* **Векторное хранилище**: ChromaDB (данные сохраняются в томе `ollama_data`).
* **Эмбеддинги**: sentence-transformers (модель all-MiniLM-L6-v2).
* **Оркестрация**: Docker Compose с изолированной сетью между ботом и движком инференса.

## Решение проблем

* **Медленная сборка**: Убедитесь, что файл `.dockerignore` исключает папки `chroma_db`, `venv` и `__pycache__`.
* **Ошибка подключения к Ollama**: Проверьте, что в файле `.env` переменная `OLLAMA_HOST` установлена как `http://ollama:11434`.
* **Ошибки прав доступа**: Если папка `chroma_db` становится недоступной на хосте, используйте команду `make clean` для сброса томов Docker.
