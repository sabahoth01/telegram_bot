## Подсказки из лекций

### Структура таблицы для логов
```sql
CREATE TABLE user_logs (
    timestamp DateTime,
    service   String,
    user_id   UInt64,
    event     String,
    object    String
) ENGINE = MergeTree()
ORDER BY (timestamp, user_id);
```

### Почему sorting key важен
- **С** ключом сортировки: бинарный поиск → очень быстро
- **Без** ключа сортировки: полное сканирование → намного медленнее
- Ключ сортировки определяет физический порядок данных на диске

### Пример подсчёта SPU

ClickHouse не поддерживает стандартные оконные функции — используйте `lagInFrame` / `leadInFrame`:

```sql
SELECT
    user_id,
    countIf(prev_ts IS NULL OR (timestamp - prev_ts) > {delta}) AS sessions
FROM (
    SELECT
        user_id,
        timestamp,
        lagInFrame(timestamp) OVER (
            PARTITION BY user_id ORDER BY timestamp
        ) AS prev_ts
    FROM user_logs
)
GROUP BY user_id;
```
