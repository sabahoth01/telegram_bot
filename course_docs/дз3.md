## Подсказки из лекций

### Типичный BigData пайплайн
```
HDFS (хранение) → Spark (обработка) → Hive (SQL-операции)
```
Затем — визуализация в Grafana или Superset.

### Зачем Docker для Spark?
- Настройка Spark на реальном кластере — сложно
- Docker Compose позволяет симулировать многоузловой кластер на одной машине
- Все компоненты (HDFS, Spark, Hive, Zeppelin) запускаются как контейнеры

### Команды запуска Spark

```bash
# Локальный режим (все ядра)
spark-submit --master local[*] your_script.py

# Режим кластера YARN
spark-submit --master yarn --deploy-mode cluster your_script.py
```
