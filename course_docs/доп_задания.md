## Дополнительное задание 1 — VirtualBox и сетевое взаимодействие

**+10 баллов**

### Задача

1. Развернуть **VirtualBox** локально
2. Установить в него **2 разных ОС**
3. На одну ОС установить **Python-сервер**
4. На другую ОС установить **web-клиент**
5. Соединить в сеть и убедиться, что клиент видит сервер

### Формат сдачи

Хорошо оформленный **пошаговый отчёт**, по которому преподаватель сможет самостоятельно воспроизвести всё с нуля.

### Что должно быть в отчёте
- Скриншоты каждого шага установки
- Конфигурация сети (тип адаптера, IP-адреса)
- Код Python-сервера
- Проверка доступности (curl / browser / ping)
- Возможные проблемы и их решения

## Дополнительное задание 2 — Telegram-бот с Gemma

**+10 баллов**

### Задача

Написать Telegram-бота, который будет иметь под капотом **открытую модель Gemma**, развёрнутую:
- в облаке (Oracle Cloud, Google Cloud, Hetzner и др.), **или**
- на локальном компьютере

### Формат сдачи

- Код бота на **GitHub** (публичный репозиторий)
- **Ссылка на бота** в Telegram (чтобы преподаватель мог его запустить и проверить)

### Минимальные требования к боту
- Бот принимает текстовые сообщения и отвечает через Gemma
- Модель реально работает (не заглушка)
- Код чистый и читаемый
- README с инструкцией по запуску

### Рекомендуемый стек
- **Ollama** — для запуска Gemma локально или на сервере
- **python-telegram-bot** — библиотека для бота
- **Docker + docker-compose** — для упаковки
- Модель: `gemma:2b` (лёгкая) или `gemma:7b` (умнее)
